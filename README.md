# CNN na classifição de nódulos pulmonares

Este repositório contém implementações de 4 modelos pré treinados de Redes Neurais Convoluacionais aplicados na tarefa de classificar nódulos pulmonares, sendo eles:
  - ResNet50
  - VGG16
  - InceptionV3
  - MobileNetV2

Sobre a Implementação:

Em problemas de classificação de imagens médicas, a dimensão restrita das bases de dados pode induzir a vieses de seleção e elevar a variabilidade dos resultados (RILEY et al., 2025). Para contornar oscilações amostrais na partição dos dados e assegurar a robustez estatística dos achados, neste trabalho foram executadas 20 repetições independentes para cada uma das arquiteturas avaliadas. A cada iteração, o conjunto de dados global é particionado de forma estratificada nas proporções de 70% para treinamento, 15% para validação e 15% para teste. Ao final das execuções, o desempenho de cada rede é mensurado pela média aritmética acompanhada do respectivo desvio-padrão para as métricas de Acurácia, Recall, F1-Score e AUC-ROC. Essa abordagem visa confirmar se as variações de desempenho decorrem da capacidade de generalização das arquiteturas ou de partições favoráveis na divisão dos dados.

Para garantir a reprodutibilidade dos experimentos diante da natureza estocástica dos algoritmos de aprendizado de máquina, adota-se o controle da aleatoriedade do sistema. O número identificador de cada uma das 20 repetições (valores inteiros de 1 a 20) é utilizado diretamente como semente (random seed). Essa configuração atua em duas frentes operacionais da implementação. Primeiro, na divisão e balanceamento dos dados, garante-se que o particionamento estratificado e os algoritmos de manipulação amostral operem sob o mesmo estado dentro de uma mesma rodada, assegurando que as quatro redes avaliadas sejam submetidas exatamente aos mesmos subconjuntos. Segundo, na otimização de hiperparâmetros, vincula-se o amostrador probabilístico do algoritmo de otimização bayesiana — que utiliza o GPSampler da biblioteca Optuna — à semente da rodada, determinando uma trajetória de exploração replicável no espaço de busca.

O pipeline experimental é estruturado em duas fases consecutivas por repetição, associando consistência metodológica e eficiência computacional. Na primeira fase, referente à calibração, o algoritmo de otimização bayesiana explora o espaço de busca delimitado ao longo de 50 iterações (trials), avaliando cada combinação candidata de hiperparâmetros por um limite de 20 épocas. O objetivo dessa etapa é identificar configurações promissoras de convergência, utilizando o F1-Score do conjunto de validação como função objetivo a ter seu erro minimizado. Na segunda fase, destinada ao treinamento final, a melhor combinação de hiperparâmetros selecionada é aplicada para reinicializar o modelo sob um treinamento estendido de até 150 épocas. Para mitigar o sobreajuste (overfitting), incorpora-se o mecanismo de interrupção precoce (early stopping) com paciência de 20 épocas. Caso o F1-Score de validação não apresente evolução nesse intervalo, o processo é encerrado e os pesos associados ao melhor estado alcançado são restaurados para a avaliação definitiva no conjunto de teste independente.
